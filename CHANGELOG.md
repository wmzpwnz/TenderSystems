# 📝 История изменений TenderSystems

## [1.0.0] - 2026-01-03 - PRODUCTION READY! 🎉

### ✨ Добавлено
- **15+ фильтров поиска** - регион, ОКПД2, цена, площадка, преимущества, авансирование
- **Реальное API автодополнения** - замена mock данных на запросы к /api/v1/search/autocomplete
- **FTP интеграция с ЕИС** - полная реализация загрузки через FTP zakupki.gov.ru
- **Celery Beat автосинхронизация** - 3 периодические задачи:
  - Синхронизация новых тендеров каждый час
  - Обновление неполных данных каждые 6 часов
  - Очистка старых тендеров ежедневно
- **Реальная статистика** на дашборде вместо хардкода
- **Health check** с реальными проверками БД и Redis
- **10 индексов БД** для производительности:
  - B-tree индексы на часто используемые колонки
  - GIN индекс для полнотекстового поиска
  - Составные индексы для комбинированных запросов
- **Модальные окна** для выбора регионов (83) и процедур (19 типов)
- **Новые поля в модели Tender**:
  - `platform` - площадка проведения торгов
  - `prepayment_type` - тип авансирования
  - `preferences` - преимущества и ограничения
  - `contract_guarantee` - обеспечение контракта

### 🔧 Исправлено
- **FTP функция** - заменена NotImplementedError на полную реализацию через EISFTPClient
- **Голые except блоки** - заменены на специфичные обработчики (ValueError, TypeError, JSONDecodeError)
- **Health check** - теперь реально проверяет БД (SELECT 1) и Redis (ping())
- **MVP ограничение** - убрано ограничение на 5 регионов, теперь все 83 региона
- **Автодополнение** - подключен реальный API вместо хардкода ['лыжи', 'мебель']

### 🗑️ Удалено
- Старый Dashboard.tsx (заменён на DashboardNew.tsx → Dashboard.tsx)
- Mock данные автодополнения
- TODO комментарии с заглушками
- Дублирующиеся маршруты в App.tsx

### 📦 Реорганизовано
- Тестовые скрипты перемещены из backend/ в backend/tests/:
  - test_eis_connection.py
  - test_parse_archive.py
  - test_soap_api.py
  - test_soap_debug.py

### 🔒 Безопасность
- **.env.example** - создан безопасный шаблон без реальных ключей
- **SECURITY.md** - полная инструкция по безопасной настройке
- **Документация** по генерации SECRET_KEY и паролей
- Предупреждения о необходимости смены дефолтных паролей

### 📚 Документация
